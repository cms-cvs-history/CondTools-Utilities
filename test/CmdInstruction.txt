setenv SCRAM_ARCH slc3_ia32_gcc323
scramv1 p CMSSW CMSSW_[VERSION]
cd CMSSW_[VERSION]/src
cvs co CondTools
cd src/CondTools/Utilities/bin
eval `scramv1 runtime -csh`

Rules:
----the name of the mapping file has the following format:
     [ObjectName]-mapping-[mappingversion].xml

1) build default mapping for the desired class with the default mapping version set to "cmsdefault". Note: this is an one time operation. Use -u option to see the help message
e.g.
./create_default_mapping EcalPedestals CondFormats/EcalObjects
will create the default mapping xml for object EcalPedestals with the name
EcalPedestals-mapping-cmsdefault.xml

1.5) cp the output xml file from step 1) to EcalPedestals-mapping-1.0.xml
modify EcalPedestals-mapping-1.0.xml
   --change to the Mapping version tag to "1.0"(!)
   --change column names and table names as you wish: *except* all ID columns have to be changed to 'IOV_VALUE_ID'
  
2) setup the offline POOL database using command setup_pool_database. Use -u option to see the help message
  --if your initial IOV is not infinity, you have to add *by hand* the column 
     "TIME NUMBER(38)" into your main class table.
   Note script: this has to be run each time new data are transfered to offline so that the pool container is up-to-date

3) build init iov and assign a name(tag) to this iov object. If no tag option is given, the default value '[objectname]-init' will be assigned to this iov.
 
Use the -h option to see the help message
e.g.
  setup_default_iov EcalPedestals CondFormatsEcalObjects sqlite_file:pippo.db EcalPedestals-mapping-1.0.xml -i 

  will assign infinite value to this IOV series and give the default name 'EcalPedestals-init' to the IOV object.
 
Other commandline tools:
 - remove_pool_tables: drop all the POOL service tables
